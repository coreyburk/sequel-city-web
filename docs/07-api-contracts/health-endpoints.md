# Health Endpoints

These endpoints expose current backend and database diagnostics for the local-first runtime.

## `GET /api/health/database`

Returns direct database connectivity status.

### Response Codes

| Status | When returned |
|---|---|
| `200` | Database connection succeeded |
| `503` | Database connection failed |

### Success And Failure Shape

The response body shape is the same for both `200` and `503`.

| Field | Type | Notes |
|---|---|---|
| `isConnected` | `boolean` | Backend authority for current connectivity result |
| `databaseName` | `string \| null` | Database name when available |
| `serverName` | `string \| null` | SQL Server name when available |
| `message` | `string` | Deterministic status message |
| `checkedAtUtc` | `string` | ISO 8601 UTC timestamp generated by the backend |

### Example `200`

```json
{
  "isConnected": true,
  "databaseName": "SequelCityCrimesDB",
  "serverName": "SEQUELCITY",
  "message": "Database connection successful.",
  "checkedAtUtc": "2026-05-05T00:00:00.000Z"
}
```

### Example `503`

```json
{
  "isConnected": false,
  "databaseName": null,
  "serverName": null,
  "message": "Database connection failed.",
  "checkedAtUtc": "2026-05-05T00:00:00.000Z"
}
```

## `GET /api/health/full`

Returns a combined API, database, and schema diagnostic payload.

### Response Codes

| Status | When returned |
|---|---|
| `200` | `data.database.isConnected` is `true` |
| `503` | `data.database.isConnected` is `false` |

### Response Shape

| Field | Type | Notes |
|---|---|---|
| `success` | `true` | Always `true` in the current implementation |
| `data.api` | `"ok"` | Indicates the Fastify API is responding |
| `data.database.status` | `"ok" \| "failed"` | Derived from database connectivity |
| `data.database.isConnected` | `boolean` | Database connectivity authority |
| `data.database.databaseName` | `string \| null` | Database name when available |
| `data.database.serverName` | `string \| null` | SQL Server name when available |
| `data.database.message` | `string` | Backend diagnostic message |
| `data.schema.status` | `"ok" \| "failed"` | Indicates schema metadata availability |
| `data.schema.tableCount` | `number` | Count of returned tables |
| `data.schema.relationshipCount` | `number` | Count of returned relationships |
| `data.schema.message` | `string` | Schema diagnostic message |

### Example `200`

```json
{
  "success": true,
  "data": {
    "api": "ok",
    "database": {
      "status": "ok",
      "isConnected": true,
      "databaseName": "SequelCityCrimesDB",
      "serverName": "SEQUELCITY",
      "message": "Database connection successful."
    },
    "schema": {
      "status": "ok",
      "tableCount": 11,
      "relationshipCount": 9,
      "message": "Schema metadata loaded successfully."
    }
  }
}
```

### Example `503`

```json
{
  "success": true,
  "data": {
    "api": "ok",
    "database": {
      "status": "failed",
      "isConnected": false,
      "databaseName": null,
      "serverName": null,
      "message": "Database connection failed."
    },
    "schema": {
      "status": "failed",
      "tableCount": 0,
      "relationshipCount": 0,
      "message": "Schema metadata unavailable."
    }
  }
}
```
